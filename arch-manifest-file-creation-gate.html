1 · Overview — Feature, Audience & Dependencies

Feature Overview

Manifest File Creation Gate enforces a hard constraint within scaffold-cli's pretool phase: files declared as targets in manifest.templates.json may only be edited if they already exist in the target project. New file creation is blocked entirely, preventing accidental generation of files outside the intended scaffold scope.

This gate protects template integrity across re-runs and ensures that only the manifest-approved file tree is ever written, no exceptions.

3 · C4 — Context & Containers

System Architecture

Gate integrates into scaffold-cli pretool phase. On scaffold generate, CLI loads manifest.templates.json, iterates all targets, checks each file's existence in the output project via fs.existsSync(), and blocks writes for non-existent files before template rendering begins.

On scaffold init, gate is disabled to allow initial project creation.

Key Interactions

  • Pack Loader → Manifest Parser: JSON parse; manifest.templates.json → targets array
  • Manifest → Gate Validator: Pass target list to gate validator
  • Gate → File System: fs.existsSync(resolvedPath) for each target
  • Gate → CLI: Either grant proceed signal or emit error with file path and reason

Migration Impact

No schema changes. Gate operates in-memory, reading only the project file tree and manifest JSON. No persistence layer required. Existing manifests are fully compatible; the gate simply enforces stricter write rules.

6 · UI/UX (React)

User Workflow

Scenario 1: Initial Scaffold (gate disabled)
1. Developer runs: scaffold init --pack v8-controller
2. Gate is disabled; all manifest targets are written to create the project
3. Project ready with all files present

Scenario 2: Later Regeneration (gate enabled)
1. Developer runs: scaffold generate --pack v8-controller
2. Gate checks all manifest targets against existing project files
3. If all exist: files are overwritten with new template renders
4. If any missing: gate blocks, error message shown, developer must restore files or reinit

8 · Deployment — Docker, Terraform, GitLab CI/CD

CI/CD Pipeline Integration

Gate is validated in CI before every npm publish. Job templates-dotnet-build-check:

  1. Runs on every push to main
  2. Scaffolds test fixtures through scaffold init + scaffold generate
  3. Compiles .NET output with dotnet build
  4. Fails if gate blocks legitimate scaffold workflows
  5. Blocks npm publish if gate validation fails

Pipeline Sequence

ESLint & TypeScript → Unit Tests → Build Check (with gate validation) → npm Publish

Build check validates that the gate does not break existing scaffold workflows. A gate violation during build-check blocks the release.

10 · Open Questions, Decisions & Risks

Open Questions

IDQuestionProposed Default
Q1Should --force flag bypass the gate?No override. Hard gate is non-negotiable. Developers restore files or reinit.
Q2Should gate apply to all packs or only v8-controller?Apply to all packs with manifest. Consistency across scaffold ecosystem.
Q3How to handle dynamic per-entity templates (e.g., {{entity}}.cs)?Treat as patterns: resolve template string to regex, check if any match existing files.
Q4Should gate validation be logged at INFO or DEBUG?DEBUG for success; INFO/WARN for violations. Violations always visible.
Q5What if manifest.templates.json is missing or malformed?Fail closed. Gate blocks all writes until manifest is valid.

Architecture Decisions

DecisionChoice + Rationale
Gate enforcement pointPretool (before render). Blocks early, avoids wasted CPU, clear error to developer immediately.
Override mechanismNo override (hard gate). Enforces invariant: manifest is source of truth. No escape hatches.
Scope: all packs or v8-controllerAll packs with manifest. Consistency, simpler code path, reduces surprises.
Existence check methodfs.existsSync (sync). Simplicity, no async complexity, negligible perf impact.

Key Risks

  • False positives: Gate blocks legitimate workflows. Mitigation: thorough CI testing, test data covering init+generate sequences.
  • Manifest out of sync: Gate blocks correct files if manifest is stale. Mitigation: CI validates manifest against template directory; build-check compiles output.
  • Developer confusion: Why can't they create new files? Mitigation: clear error messages, README explanation, example workflow.
  • Windows path separator issues: Gate passes on Linux/macOS but fails on Windows. Mitigation: normalize paths to posix format; test all three OSes.
