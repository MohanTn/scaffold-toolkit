Functional Requirements
IDRequirementAcceptance Criteria
FR1Generate OpenAPI 3.0 spec from controllersSwagger UI at /swagger; spec at /swagger/v1/swagger.json and /swagger/v2/swagger.json
FR2Support semantic API versioning (v1, v2 routes)Same entity entity scaffolded to /api/v1/{entity} and /api/v2/{entity}; routing middleware selects correct handler
FR3Per-version controller definitionsEach version has separate controller file (EntityControllerV1.cs, EntityControllerV2.cs); can differ in response DTOs, validation, HTTP status codes
FR4EFCore value converters for encrypted/encoded fieldsManifest author declares field option: { 'encrypt': true }; template injects ValueConverter(encrypt, decrypt) into EntityConfiguration
FR5Shadow property support in EFCoreConfiguration template supports shadow properties (e.g., CreatedAt, UpdatedAt); queries can include them without exposing in entity class
FR6OpenAPI descriptions per versionSwagger operation summaries differ between v1 and v2; deprecated fields marked in v1 with 'deprecated: true' in OpenAPI schema
Non-Functional Requirements
AttributeMeasurable Target
PerformanceValue converter overhead <1ms per entity; shadow properties add no query cost vs. explicit columns
SecurityEncrypted fields are never exposed unencrypted in logs, DTOs, or exception traces; value converter uses DPAPI or AES-256
MaintainabilityVersioning and encryption logic in base templates; per-entity scaffolded code requires no duplication
Compatibilityv1 routes remain active indefinitely; v2 routes coexist; no breaking changes to existing clients until v1 EOL announced
DocumentationSwagger UI updates automatically; no manual OpenAPI YAML editing required

Schema Migration Strategy

EFCore enhancements impact:

  • Value Converters: No schema change; existing column persists as plaintext in DB, decrypted in memory. On opt-in, developer manually creates migration to re-encrypt existing rows.
  • Shadow Properties: No EFCore annotation in entity class; automatically generated columns (e.g., CreatedAt) via configuration. Migration creates columns silently.
  • Per-Version DTOs: No DB change; application layer only. Clients see different response shapes based on route version.

10 · Open Questions, Decisions & Risks

Open Questions, Decisions & Risks

Open Questions
IDQuestionWhy It MattersProposed DefaultStatus
OQ1Which value converter algorithm (DPAPI vs. AES-256) for encrypted fields?DPAPI ties encryption to the machine; AES-256 is portable across servers and containers.Use AES-256 with key rotation in AppSettings. Encode as Base64 in DB.Open
